# Курсовой проект по курсу «Операционные системы»

## Цель курсового проекта
1. Приобретение практических навыков в использовании знаний, полученных в течение курса.  
2. Проведение исследования в выбранной предметной области.

## Задание
Необходимо спроектировать и реализовать программный прототип в соответствии с выбранным вариантом. Произвести анализ и сделать вывод на основании данных, полученных при работе программного протипа.

---

## Вариант 23: Клиент-серверная система для передачи мгновенных сообщений с групповыми чатами

### **Базовый функционал (общий для вариантов 22–30):**
- Клиент может присоединиться к серверу, введя логин.
- Клиент может отправить сообщение другому клиенту по его логину.
- Клиент в реальном времени принимает сообщения от других клиентов.

### **Специфика варианта 23:**
- **Дополнительная возможность:** создание «групповых чатов».
- **Технология связи:** связь между сервером и клиентом должна быть реализована при помощи **очередей сообщений** (например, **ZeroMQ**).

### **Требования к реализации:**
1. Реализовать серверную часть, координирующую подключение клиентов и обработку сообщений.
2. Реализовать клиентскую часть с возможностью:
   - подключения к серверу по логину;
   - отправки личных сообщений другим клиентам;
   - участия в групповых чатах (создание, присоединение, обмен сообщениями).
3. Использовать механизм очередей сообщений (например, ZeroMQ) для обмена данными между сервером и клиентами.
4. Обеспечить реальное время получения сообщений.

### **Отчёт должен включать:**
1. Проектирование системы (архитектура, схема взаимодействия).
2. Исходный код сервера и клиента.
3. Описание процесса тестирования.
4. Анализ работы системы (производительность, надёжность, возможные проблемы).
5. Выводы по проделанной работе.

---

## Примечания
- Для варианта 23 **не требуется** реализовывать хранение истории переписок, поиск по ним или отправку отложенных сообщений — это задачи других вариантов.
- **ZeroMQ** указан как пример, можно использовать другую библиотеку очередей сообщений, но необходимо обосновать выбор.
- Система должна быть **многопользовательской** и работать в **сетевом режиме**.
